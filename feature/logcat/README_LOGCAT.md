# feature/logcat — Логкат

Потоковый просмотр системных логов устройства с фильтрацией, поиском и настройкой отображения.

## Функциональность

### Стриминг логов

**Команда:**
```
adb logcat [-v <format>] [filter-specs...]
```

- Кнопка **Start** — запуск стриминга, начало накопления записей
- Кнопка **Stop** — остановка стриминга (записи остаются)
- **Clear** — очистка буфера записей в памяти

**Буфер:** ограниченное количество записей (старые вытесняются при переполнении).

---

### Фильтрация

#### По уровню лога

| Уровень | Символ | Описание |
|---|---|---|
| Verbose | V | Все сообщения |
| Debug | D | Отладочные и выше |
| Info | I | Информационные и выше |
| Warning | W | Предупреждения и ошибки |
| Error | E | Только ошибки |
| Fatal | F | Только критические |

#### По тегу
- Текстовый ввод тега для фильтрации (подстрока или точное совпадение)

#### По пакету
- Фильтр по package name — показывать только логи от конкретного приложения
- Может предустанавливаться при переходе из `feature/packages`

#### Поиск по тексту
- Поиск по содержимому сообщения (substring match)
- Подсветка совпадений

---

### Настройки отображения

#### Режим отображения
- **Compact** — одна строка на запись: уровень + тег + сообщение
- **Full** — многострочный вид с PID, TID, временем, полным сообщением

#### Шрифт
- Выбор семейства шрифта (моноширинные: JetBrains Mono, Cascadia, System Mono, …)
- Размер шрифта

#### Живой поток
- Опция **Smooth live stream** включает по-кадровую публикацию новых строк (около 16 мс между кадрами).
- Для защиты от отставания при бурстах используется адаптивный размер чанка публикации и ускоренный drain backlog.

---

### Запись лога

**Формат строки логкат:**
```
<дата> <время> <PID> <TID> <уровень> <тег>: <сообщение>
```

Пример:
```
03-10 14:22:01.345  1234  5678 I ActivityManager: Start proc com.example for activity
```

---

### Дополнительно
- Автопрокрутка вниз при новых записях (toggle)
- Статусбар: количество записей в буфере, статус стриминга (Running / Stopped)
- Плавное смещение элементов списка при добавлении новых строк (`animateItem`).

## Архитектура модуля

```
feature/logcat/
├── LogcatComponent             — публичный интерфейс: start/stop, filter, display settings
├── DefaultLogcatComponent      — стриминг через ProcessRunner, управление буфером
├── LogcatState                 — Running status, буфер записей, фильтры,
│                                 LogcatDisplayMode (COMPACT/FULL),
│                                 LogcatFontFamily, шрифт
├── LogcatError                 — типы ошибок
├── LogcatDisplayMode           — COMPACT / FULL
├── LogcatFontFamily            — доступные шрифты
└── ui/
    ├── LogcatScreen            — основной layout
    ├── LogcatSettingsPanel     — панель настроек фильтрации и отображения
    ├── LogcatEntryRow          — строка / блок одной записи лога
    └── LogcatStatusBar         — статусбар: счётчик, статус
```

## Связи с другими модулями
- Из `feature/packages`: кнопка «Track in Logcat» → предустанавливает фильтр по package name.
- Из `feature/notifications`: переход с фильтром по пакету уведомления.
