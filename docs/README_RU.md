# ADB Deck

🇷🇺 Русский | 🇬🇧 [English](../README.md)

---

ADB Deck — десктопный GUI для работы с Android-устройствами через ADB.
Удобный интерфейс поверх сырых `adb`-команд — не нужно помнить синтаксис shell или держать открытыми пять вкладок терминала.

Создан для **Android-разработчиков**, **QA-инженеров** и всех, кто набирает `adb shell` чаще, чем хотелось бы.

---

## 🚀 Возможности

### 📱 Менеджер устройств
Подключение и переключение между несколькими устройствами одновременно — по USB и Wi-Fi. Запоминает ранее подключённые сетевые устройства для быстрого переподключения.

### 📋 Logcat
Живой поток логов с фильтрацией по тегу, пакету и уровню. Цветная подсветка, компактный и полный форматы, плавный скролл в реальном времени. Импорт и экспорт сессий.

### 📦 Менеджер пакетов
Просмотр всех установленных приложений с фильтрами: Пользовательские / Системные / Отключённые / Отлаживаемые. Действия для каждого приложения: запуск, остановка, системная информация, экспорт APK, очистка данных, удаление. Детальный просмотр: версия, target SDK, min SDK, даты установки и обновления. Переход в Logcat с фильтром по пакету в один клик.

### 📂 Файловый проводник
Двухпанельный вид — локальная файловая система слева, файловая система устройства справа. Отправка файлов на устройство, скачивание с него. Создание папок, переименование, удаление.

### 🖥️ Инструменты экрана
Снимки экрана с настраиваемым путём сохранения и форматом (PNG/JPEG). Мгновенный предпросмотр, копирование в буфер обмена или открытие папки. Запись видео с экрана.

### 🪞 Зеркалирование экрана (Scrcpy)
Запуск scrcpy прямо из приложения с панелью настроек: полноэкранный режим, без рамки, поверх всех окон, размер окна, видеокодек, режим клавиатуры и мыши.

### 📲 Установка APK
Установка APK через drag-and-drop или выбор файла — на подключённое устройство.

### 🔗 Deep Links и Intent
Запуск диплинков и кастомных интентов без терминала. Задавайте action, package, component и category. Переключение между режимами Deep Link, Intent и raw shell-командой. История и переиспользуемые шаблоны.

### 🔔 Инспектор и конструктор уведомлений
Живая лента уведомлений с историей и сохранёнными снимками. Детальный просмотр: пакет, канал, важность, флаги, действия, визуальные параметры, raw dump. Встроенный конструктор для тестирования собственных уведомлений.

### ℹ️ Информация об устройстве
Подробные сведения по 13 разделам: Обзор, Сборка, Дисплей, CPU / RAM, Батарея, Сеть, Сотовая связь, Модем, IMS/RCS, Хранилище, Безопасность, Система. Экспорт всего в JSON.

### ⚡ Быстрые переключатели
Включение и выключение Wi-Fi, мобильных данных, Bluetooth, режима полёта и «Не выключать экран» прямо из интерфейса. Настройка масштабов анимации (окна, переходов, аниматора) через слайдер — удобно для отключения анимаций во время UI-тестирования.

### 📊 Системный монитор
Список процессов в реальном времени с живыми графиками CPU и памяти. Детали по каждому процессу: RSS, heap, VSZ, потоки, файловые дескрипторы, состояние. Kill или force-stop процесса, или переход к его записи в менеджере пакетов.

### 👤 Контакты
Просмотр контактов устройства.

### 📁 Файловая система
Обзор дерева файловой системы устройства.

### 🪵 ADB Лог
Каждая ADB-команда, которую отправляет приложение, фиксируется во встроенной панели лога — удобно для изучения, аудита или отладки неожиданного поведения.

## ⚙️ Установка

1. Скачайте последний релиз `.dmg` / `.deb` / `.msi` из раздела [Releases](../../../releases)
2. Запустите **ADB Deck**
3. Откройте **Settings**
4. Укажите путь к бинарнику `adb` (из [Android Platform Tools](https://developer.android.com/tools/releases/platform-tools))
5. Нажмите **Check**

Всё. После этого терминал больше не нужен.

## 🔧 Требования

- **ADB** — [Android Platform Tools](https://developer.android.com/tools/releases/platform-tools)
- Android-устройство с включённой **отладкой по USB** (Параметры разработчика)
- Для Wi-Fi отладки: Android 11+ (беспроводная отладка) или `adb tcpip` на Android 10 и ниже
- **Scrcpy** — нужен только для зеркалирования экрана (устанавливается отдельно)

**Поддерживаемые платформы:**
- macOS
- Linux
- Windows

## 🏗️ Архитектура

| Слой | Технология |
|---|---|
| UI | Kotlin + Compose Multiplatform (Desktop) |
| Навигация | Decompose (child stack, lifecycle компонентов) |
| DI | Koin |
| Асинхронность | Coroutines + StateFlow |
| Персистенция настроек | Kotlinx Serialization → `~/.adbdeck/settings.json` |

**Структура модулей:**

```
:app                  — точка входа, DI, корневая навигация
:core:process         — ProcessRunner (обёртка над adb subprocess)
:core:adb-api         — интерфейсы и доменные модели
:core:adb-impl        — выполнение ADB-команд и парсинг вывода
:core:settings        — настройки приложения (путь к adb, тема)
:core:designsystem    — цвета, типографика, отступы
:core:ui              — переиспользуемые composable (loading, error, banners)
:core:utils           — расширения
:feature:*            — один модуль на каждый экран, полная изоляция
```

Каждый feature-модуль владеет своим состоянием, интерфейсом компонента, реализацией и UI. Фичи не зависят друг от друга — кросс-фичевая навигация идёт через корневой компонент через callback-ы.

## ⚠️ Дисклеймер

ADB — мощный инструмент. Он может очистить данные приложения, удалить системные пакеты, убить процессы и сделать другие вещи, которые нельзя отменить.

Приложение не добавляет защитных ограничений.
Если не знаешь, что делает кнопка — лучше не нажимай.
